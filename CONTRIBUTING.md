# Contributing to Janta Electricals

Thank you for your interest in helping improve Janta Electricals' online presence. Your contributions are valuable to us.

## Code of Conduct

This project and everyone participating in it is governed by our [Code of Conduct](CODE_OF_CONDUCT.md). By participating, you are expected to uphold this code. Please report unacceptable behavior.

## Before you contribute

- Use the information in `data/business.json` as the single source of truth for business details.
- Do not publish customer contact details, project photos, or unverified claims without the business owner's explicit approval.
- Keep service descriptions clear, accurate, and locally relevant to Bhowra, Dhanbad, Jharkhand.

To get the project running on your local machine for development and testing purposes, please follow the instructions in our README.md.

## How to Contribute

1.  **Find an issue:** Look for an issue that interests you in our Issues tab. We recommend starting with issues labeled `good first issue`.
2.  **Fork & Clone:** Fork the repository to your own GitHub account and then clone it to your local machine.
3.  **Create a branch:** Create a new branch for your changes. Please use a descriptive name, like `feat/add-service-gallery` or `fix/correct-contact-number`.
    ```bash
    git checkout -b <branch-name>
    ```
4.  **Make your changes:** Edit the code to address the issue. Remember to keep business facts consistent across all pages.
5.  **Test your changes:** Ensure that the site works as expected. Test all links and view the site at both desktop and mobile screen widths using your browser's developer tools.
6.  **Commit your work:** Write a clear, concise commit message.
    ```bash
    git commit -m "feat: Add gallery to services page"
    ```
7.  **Push to your fork:**
    ```bash
    git push origin <branch-name>
    ```
8.  **Open a Pull Request:** Go to the original repository on GitHub and open a pull request. Provide a concise summary of your changes and include screenshots for any visual updates. Link the pull request to the issue it resolves.

## Reporting an issue

We use GitHub Issues to track bugs, content corrections, and feature ideas.

- **Report a Bug**
- **Suggest a Content Correction**
- **Propose a New Feature**

**Important:** Never include sensitive customer information in a public issue.
